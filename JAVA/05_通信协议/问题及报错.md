
## 问题1 getDisplayMedia函数未定义 
1、控制台报错：uncaught (in promise) TypeError: Cannot read properties of undefined (reading 'getDisplayMedia')
2、分析：
	谷歌浏览器使用 navigator.mediaDevices.getUserMedia获取资源时报错，
3、解决方法：
	1-浏览器输入
	chrome://flags/#unsafely-treat-insecure-origin-as-secure
	2-开启Insecure origins treated as secure
		在下方输入栏内输入，你访问的地址url，然后将右侧Disabled 改成 Enabled即可
	3-浏览器重启

4、原文地址：[Chrome 浏览器获取音频设备权限问题 Cannot read property getUserMedia of undefined_chrome getusermedia_GeneralMonkey的博客-CSDN博客](https://blog.csdn.net/QT_Port/article/details/106860099)

## 局域网连接，ws直接断开
如果您在局域网中使用 WebRTC 进行视频聊天，出现了“WebSocket 断开连接”的现象，可能是由于如下原因：

1.  网络不稳定，导致连接不可靠。在 WebRTC 连接的建立和维护过程中，网络环境的稳定性对连接的质量起着至关重要的作用。如果局域网的网络环境不稳定，会导致 WebRTC 的连接不可靠，并出现 WebSocket 断开连接的现象。
    
2.  防火墙或 NAT 转发等网络限制。在使用 WebRTC 进行连接时，需要保证网络中的所有设备都能够连接到服务器。如果防火墙或NAT转发等网络限制，就可能出现连接的问题。
    
3.  浏览器或系统不兼容。如果使用的浏览器或操作系统不支持 WebRTC，会导致连接不成功，从而无法进行视频聊天。
    

解决方法：

1.  稳定的网络环境是保障 WebRTC 连接的前提条件，建议先确保局域网内的网络环境稳定。如果局域网的网络环境不稳定，可以考虑增加网络带宽，或使用有线网络等方式来提高网络连接质量。
    
2.  检查防火墙和 NAT 转发等网络限制是否存在，并进行相应的调整。
    
3.  确认使用的浏览器和操作系统是否支持WebRTC。如果不支持，可以考虑升级浏览器或操作系统，或者切换使用支持 WebRTC 的浏览器。
